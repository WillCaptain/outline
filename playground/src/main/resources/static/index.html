Outline A structurally-typed language powered by GCP inference
Most typed languages make you write types so the compiler can check them.
Outline flips this: the compiler figures out every type from how you use values —
down to field names, HOF chains, and multi-level structural constraints.
What makes it different
🦆
Duck Typing with Iron-Clad Types
Structural typing means you write values, not interfaces. GCP synthesises the full entity type from how functions access fields — zero declarations, zero annotations, zero boilerplate. Extra fields are silently allowed; missing fields are caught at compile time with a precise error pointing to the exact call site.
🔀
Backward Constraint Propagation
Type constraints flow backwards through HOF chains. When is_passing needs score ≥ 60, GCP propagates that constraint back through lift(get_score) to determine the shape of the original entity — a level of inference no other practical language achieves without explicit generic parameters.
🔁
Future this — Self-Type Extension Chaining
Methods that return this always resolve to the concrete receiver type, not the declared base type. Extend a base entity, call its method, and the sub-entity's methods are still accessible on the result — covariant self-type without F-bounded generics or Self annotations.
⚡
Full Dynamic Inference, No Limits
Church numerals, generic HOF combinators, deeply nested entity chains — GCP infers it all. The type system handles arbitrary complexity: generic entities with type parameters, Poly union types from branch divergence, literal types that are their value, and default fields that auto-fill at construction.
